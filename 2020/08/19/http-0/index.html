[HTTP]0-历史 定义
Created 2020-08-19| Updated 2020-08-23
Post View:

本篇总结来自极客时间的透视HTTP协议

历史

  • 20世纪60年代，美国国防部高等研究计划署（ARPA）建立了ARPA网，它有四个分部在各地的节点，被认为是如今互联网的“始祖”。

  • 70年代，基于对ARPA网的实践和思考，研究人员发明出了著名的TCP/IP协议。TCP/IP协议由于具有良好的分层结构和稳定的性能，迅速战胜其他对手流行起来，在80年代中期进入了UNIX系统内核。

  • 1989年，HTTP诞生。Tim Berners Lee发表了一篇论文提出了在互联网上构建超链接文档系统的构想。这篇文章中他确立了三项关键技术

    • URI：统一资源标识符，作为互联网上资源的唯一身份
    • HTML：超文本标记语言，描述超文本文档
    • HTTP：超文本传输协议，用来传输超文本

  • 20世纪90年代初期，HTTP被定义为0.9版，结构简单，为了便于服务和客户端处理，它采用了纯文本格式。只允许使用“GET”动作从服务器上获取HTML文档，并且在响应请求之后关闭连接。功能有限但作为一个prototype验证了web服务的可行性。

  • 1996年，在一些列新技术的推动下（Mosaic Apache JPEG MP3），HTTP/1.0版本正式发布。对0.9版本进行增强，和现在的HTTP差别不大了。

    • 增加了HEAD, POST方法等
    • 增加了响应状态吗，标记可能的错误原因ß
    • 引入了协议版本号的概念
    • 引入了HTTP头部的概念(Header)，让HTTP处理请求和响应更加灵活。
    • 传输的数据不再仅限于文本

  • HTTP/1.0并不是一个“标准”，只是记录已有时间和模式的一份参考文档，不具有实际的约束力，相当于一个备忘录。对于当时正在蓬勃发展的互联网来说并没有太大的意义，各方势力仍然按照自己的意图继续在市场上奋力拼杀。

  • 1995年网景的Netscape Navigator和微软的IE开始浏览器大战。它再一次极大推动了Web的发展，在这个过程中经受了实战检验，于是在“浏览器大战”结束之后的1999年，HTTP/1.1发布了RFC文档，编号为2616，正式确立了延续十余年的传奇。

  • HTTP/1.1是对1.0的小幅修正，但重要区别是：它是一个正式的标准，而不是一份可有可无的参考文档。它主要的变更点有：

    • 增加了PUT、DELETE等新方法
    • 增加了缓存管理和控制
    • 明确了连接管理，允许持久连接
    • 允许响应数据分块（chunked），利于传输大文件
    • 强制要求Host头，让互联网主机托管成为可能。

  • 由于HTTP/1.1过于庞大和复杂，在2014年又做了一次修订，原来的一个大文档被拆分成了6份较小的文档，编号为7230-7235，优化了一些细节。

  • 2015年，发布HTTP/2，RFC编号7540。Google开发了自己的浏览器Chrome，推出了新的SPDY协议，借用户占有率优势把SPDY推上标准宝座。

  • HTTP/2充分考虑了现金互联网显壮：带宽、移动、不安全。主要特点有：

    • 二进制协议，不再是纯文本
    • 可发起多个请求，废弃了1.1里的管道
    • 使用专用算法压缩头部，减少数据传输量
    • 允许服务器主动向客户推送数据
    • 增强了安全性，“事实上”要求加密通信

  • HTTP/2目前普及率较低

  • 2018， HTTP/3正式进入标准化制定阶段。

HTTP是什么

  • Hyper Text Transfer Protocol 超文本传输协议

协议

  • HTTP是一个用在计算机世界里的协议。它使用计算机能够理解的语言确立了一种计算机之间交流通信的规范，以及相关的各种控制和错误处理方式。

传输

  • HTTP是双向协议，分请求方和应答方/响应方
  • HTTP是一个在计算机世界里专门用来在两点之间传输数据的约定和规范

超文本

  • 超文本就是超越了普通文本的文本，它是文字、图片音频和视频等的混合体。
  • 含有超链接，能够从一个“超文本”跳跃到另一个“超文本”。
  • HTTP是一个在计算机世界里专门在两点之间传输文字、图片、音频、视频等超文本数据的约定和规范。

HTTP不是什么

  • HTTP不是互联网。但它构建互联网的一块重要的拼图。
  • HTTP不是编程语言。编程语言是人与计算机沟通交流所使用的语言，而HTTP是计算机之间沟通交流的语言。我们无法使用HTTP来编程，但可以用编程语言去实现HTTP。
  • HTTP不是HTML。HTML是超文本的载体，是一种标记语言。HTTP传输最多的可能就是HTML。
  • HTTP不是一个孤立的协议。在互联网世界里，HTTP通常跑在TCP/IP协议栈之上，依靠IP协议实现寻址和路由、TCP协议实现可靠数据传输、DNS协议实现域名查找、SSL/TLS协议实现安全通信。此外还有一些协议依赖于HTTP，例如WebSocket、HTTPDNS等。这些协议相互交织构成协议网，而HTTP则处于中心地址。
