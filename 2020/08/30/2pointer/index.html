11. 盛最多水的容器

给你 n 个非负整数 a1，a2，…，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂直线 i 的两个端点分别为 (i, ai) 和 (i, 0)。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

说明：你不能倾斜容器，且 n 的值至少为 2。

思路：

  • 从数组两头开始分别两个指针
  • 保存最大面积
  • 向内移动当前对应数值比较小的指针
  • 当两个对应数值相等时，默认移动左边（右边也可）

16. 最接近的三数之和

搞了两天 天啦撸

给定一个包括 n 个整数的数组 nums 和 一个目标值 target。找出 nums 中的三个整数，使得它们的和与 target 最接近。返回这三个数的和。假定每组输入只存在唯一答案。

思路：

  1. 给数组从小到大排序
  2. 设定3个指针，分别指向数组的头(pA)和尾(pC)，还有一个初始指向pA的下一个(pB)
  3. 设定合sum，绝对值abs
  4. 条件：
    1. 如果和大于target，移动pC
    2. 如果和小于target，先移动pB。pB到pC-1时移动pA并且恢复pB到pA的下一个
    3. 移动的同时判断sum和abs

执行用时：96 ms, 在所有 JavaScript 提交中击败了38.78%的用户

内存消耗：38.3 MB, 在所有 JavaScript 提交中击败了51.23%的用户

80. 删除排序数组中的重复项 II

给定一个排序数组，你需要在原地删除重复出现的元素，使得每个元素最多出现两次，返回移除后数组的新长度。

不要使用额外的数组空间，你必须在原地修改输入数组并在使用 O(1) 额外空间的条件下完成。

  1. splice这么好用，为啥还要用双指针XD（其实因为双指针搞了半天没搞出来）

执行用时：92 ms, 在所有 JavaScript 提交中击败了53.88%的用户

内存消耗：39.7 MB, 在所有 JavaScript 提交中击败了88.17%的用户

  1. 双指针法

思路：

  • 我们使用了两个指针，i 是遍历指针，指向当前遍历的元素；j 指向下一个要覆盖元素的位置。
  • 同样，我们用 count 记录当前数字出现的次数。count 的最小计数始终为 1。
  • 我们从索引 1 开始一次处理一个数组元素。
  • 若当前元素与前一个元素相同，即 nums[i]==nums[i-1]，则 count++。若 count > 2，则说明遇到了多余的重复项。在这种情况下，我们只向前移动 i，而 j 不动。
  • 若 count <=2，则我们将 i 所指向的元素移动到 j 位置，并同时增加 i 和 j。
  • 若当前元素与前一个元素不相同，即 nums[i] != nums[i - 1]，说明遇到了新元素，则我们更新 count = 1，并且将该元素移动到 j 位置，并同时增加 i 和 j。
  • 当数组遍历完成，则返回 j。

执行用时：96 ms, 在所有 JavaScript 提交中击败了43.78%的用户

内存消耗：39.6 MB, 在所有 JavaScript 提交中击败了93.75%的用户
